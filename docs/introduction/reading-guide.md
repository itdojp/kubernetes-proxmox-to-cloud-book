## Kustomize→Helm の学習順（理由）

- Kustomize: Kubernetes のリソース構造を理解した上で、差分（overlay）を扱う
- Helm: テンプレートと release 管理で運用をスケールさせる（ただしアンチパターンもある）

本書では、概念理解（Kustomize）→ パッケージ運用（Helm）として整理します。
