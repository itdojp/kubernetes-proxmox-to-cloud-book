# 更新履歴とメンテ方針

## この章の学習目標（3〜5）

- 更新履歴（CHANGELOG）への導線を把握できる
- 更新時に何を優先して確認すべきか理解できる

## 更新履歴

- `CHANGELOG.md` を参照してください: https://github.com/itdojp/kubernetes-proxmox-to-cloud-book/blob/main/CHANGELOG.md

## メンテ方針（ドラフト）

- 誤字脱字、参照リンク修正: 随時
- 手順・運用の更新: 影響範囲（検証/本番）と破壊的変更の有無を明記する
- バージョン更新: 付録 Version Matrix と該当章を同時に更新する

## まとめ

更新は「再現性」と「安全性」を最優先に行います。
